# Agile, Scrum y Kanban

## ¿Qué es Agile?
Agile es una forma de trabajar en proyectos, especialmente en desarrollo de software, basada en la entrega progresiva de valor, la adaptación al cambio y la colaboración continua. En lugar de planificar todo desde el principio de manera rígida, Agile propone avanzar por fases pequeñas, revisando y mejorando el producto constantemente.

Su objetivo principal es desarrollar software útil de forma flexible, respondiendo a los cambios y teniendo en cuenta las necesidades reales del usuario.

## ¿Qué es Scrum?
Scrum es un marco de trabajo dentro de Agile. Se basa en dividir el proyecto en periodos cortos llamados sprints, que suelen durar entre una y cuatro semanas. En cada sprint se planifica una parte del trabajo, se desarrolla y se revisa el resultado.

### Roles principales en Scrum
- Product Owner: define las necesidades del producto y prioriza el trabajo
- Scrum Master: facilita el proceso y ayuda al equipo a seguir Scrum
- Equipo de desarrollo: construye el producto

### Elementos principales de Scrum
- Product Backlog: lista de tareas o funcionalidades pendientes
- Sprint Backlog: conjunto de tareas elegidas para un sprint
- Sprint: periodo de trabajo corto
- Review: revisión del trabajo realizado al final del sprint

## ¿Qué es Kanban?
Kanban es otra metodología de trabajo ágil centrada en la organización visual de tareas. Se suele representar con columnas como “Pendiente”, “En proceso” y “Hecho”. Cada tarea avanza por esas columnas según su estado.

Kanban permite ver fácilmente qué se está haciendo, qué queda por hacer y si hay bloqueos.

## Diferencias entre Scrum y Kanban
Aunque ambos pertenecen al enfoque Agile, tienen diferencias claras.

### Scrum
- trabaja por sprints
- tiene roles definidos
- planifica bloques de trabajo cerrados
- es útil cuando se quiere una estructura más marcada

### Kanban
- flujo continuo de trabajo
- no necesita sprints
- no exige roles específicos
- es útil para organizar tareas de forma visual y flexible

## ¿Cuándo usar Scrum?
Scrum puede ser útil cuando el equipo trabaja en un proyecto con objetivos más definidos y quiere dividir el trabajo en periodos concretos. También es buena opción cuando se necesita una planificación más estructurada.

## ¿Cuándo usar Kanban?
Kanban es útil cuando se quiere visualizar el trabajo y adaptarse continuamente a nuevas tareas o cambios. Es especialmente práctico para proyectos pequeños o para organizar el día a día de forma sencilla.

## Conclusión
Agile es una filosofía de trabajo flexible centrada en la mejora continua. Dentro de ella, Scrum y Kanban son dos formas distintas de organizar el desarrollo. Scrum ofrece una estructura basada en sprints, mientras que Kanban propone un flujo visual y continuo. Para este proyecto, Kanban resulta especialmente útil para organizar las tareas de desarrollo de manera clara.
